1) Bootstrapping & Config Resolution

Task awal untuk menyiapkan path input, env, parser argumen, dan mode eksekusi.

Menerima Dari

CLI user dan environment runtime.

Yang Diproses

Argumen run, env key, mode retrieval, dan pilihan LLM atau fallback.

Mengirim Ke

Task 2 sebagai konfigurasi run final.

Input

  • Argumen CLI (--no-llm, --retrieval-mode, --output)
  • Env GROQ_API_KEY, MODEL_NAME, GOOGLE_API_KEY

Proses

  • Load env: load_dotenv()
  • Sanitasi proxy: _disable_blackhole_proxies()
  • Pilih translator: _build_translator()

Output

  • Konfigurasi run final
  • Status translator: LLM aktif / fallback

File Terkait

  • run_reasoning_translation.py
  • reasoning_llm_translator.py

Fungsi Terkait

  • build_parser(), run_pipeline(), _build_translator()
  • LLMReasoningTranslator.__init__()

2) Ingest Input & Flatten Source Facts

Task normalisasi metadata dari document_metadata.payload menjadi daftar fakta per field.

Menerima Dari

Task 1 dan file document_metadata.payload.

Yang Diproses

Nested JSON menjadi field-level fact yang punya path, value, dan source fact text.

Mengirim Ke

Task 4 sebagai antrian SourceFact per field.

Input

  • document_metadata.payload
  • Objek bertingkat: typography, page_layout, spacing, dst

Proses

  • Load JSON: _load_json()
  • Deteksi jenis input: _detect_input_kind()
  • Flatten field: _flatten_output_parameters()
  • Bentuk fakta natural-language: _output_source_fact()

Output

  • List SourceFact per field
  • Contoh: typography.font_size_body_pt = 12

File Terkait

  • run_reasoning_translation.py
  • reasoning_schema.py

Fungsi Terkait

  • _build_source_facts(), _output_source_fact()
  • Dataclass: SourceFact

4) Retrieval Per Fact (Lexical / Embedding)

Task pengambilan konteks dictionary yang relevan untuk masing-masing SourceFact.

Menerima Dari

Task 2 (SourceFact) dan Task 3 (catalog chunks).

Yang Diproses

Query fakta per field untuk mencari chunk property python-docx yang paling dekat.

Mengirim Ke

Task 5 sebagai retrieval context untuk translation candidate.

Input

  • 1 buah SourceFact
  • Catalog chunks
  • Mode retrieval: lexical / embedding

Proses

  • Query facts: _query_for_fact()
  • Lexical scoring: _tokenize(), _lexical_score()
  • Embedding scoring (opsional): _cosine_similarity()

Output

  • RetrievalOutcome(status, chunks, error)
  • Top-K chunk kandidat target path

File Terkait

  • run_reasoning_translation.py

Fungsi Terkait

  • Kelas: DictionaryRetriever
  • retrieve(), _retrieve_lexical(), _retrieve_embedding()

5) Translation Candidate (LLM / Rule Fallback)

Task menerjemahkan fakta ke kandidat mapping python-docx dengan decision steps audit.

Menerima Dari

Task 4: SourceFact dan top-k retrieved chunks.

Yang Diproses

Hubungan antara fakta sumber, target path, normalized value, confidence, dan audit reasoning.

Mengirim Ke

Task 6 sebagai ReasoningCandidate yang belum final.

Input

  • SourceFact
  • Retrieved dictionary chunks

Proses

  • LLM path: LLMReasoningTranslator.translate()
  • Fallback path: _fallback_candidate()
  • Guard error/rate-limit: _safe_error_detail(), _is_rate_limit_error()

Output

  • ReasoningCandidate dengan status mapped/unmapped/needs_review
  • Target path, normalized value, confidence, reason

File Terkait

  • reasoning_llm_translator.py
  • run_reasoning_translation.py

Fungsi Terkait

  • translate(), _extract_json_object()
  • _margin_target_path(), _first_available_path()

Output

  • Status final: accepted/rejected/unmapped/needs_review/retrieve_failed

File Terkait

Fungsi Terkait

  • validate_reasoning_candidate()
